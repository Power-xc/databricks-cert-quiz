🧠 시험 직전 핵심 암기 노트
문제의 신호를 보고 가장 가까운 답의 방향을 빠르게 잡는 표
암기 공식은 1차 분류용이야. 문제에 비용, 지연 시간, 권한, 데이터 수명 주기 같은 추가 조건이 있으면 그 조건을 우선해야 해.

🏗️ 아키텍처와 저장

데이터 레이크와 웨어하우스의 장점을 결합한 아키텍처→ Data Lakehouse
ACID, 트랜잭션 로그, 스키마 적용, Time Travel→ Delta LakeLakehouse의 신뢰성 있는 테이블 저장 계층
다양한 형식의 대규모 원천·원시 데이터 저장→ Data Lake / BronzeLakehouse 문제라면 원시 수집 계층은 보통 Bronze
BI, 리포팅, 비즈니스 집계, 고도로 정제된 데이터→ Data Warehouse 개념 / GoldLakehouse의 분석 제공 계층을 묻는다면 Gold

🖥️ 컴퓨트 선택

대화형 노트북 개발, 탐색, 임시 분석→ All-purpose compute
예약 작업, 워크플로, 프로덕션 자동 실행→ Jobs compute프로덕션 Job에 all-purpose compute는 권장되지 않음
SQL 쿼리, 대시보드, BI 도구 연결→ SQL warehouse
인프라 관리 없이 자동 시작·확장·종료→ Serverless compute
Unity Catalog 데이터에 접근할 컴퓨트 조건→ UC 호환 access modeSQL warehouse는 항상 Unity Catalog를 지원. 별도 컴퓨트 종류를 뜻하는 것은 아님

🥉 Medallion 계층

원본 보존, 최소 변환, 재처리 가능한 수집 데이터→ Bronze
정제, 검증, 중복 제거, 조인, 표준화→ Silver
비즈니스 집계, KPI, 대시보드·리포팅용 데이터→ Gold
신뢰 가능한 공통 상세 데이터와 최종 집계의 위치→ 상세·정합 데이터는 Silver, 집계는 Gold

🗃️ 테이블 종류

Databricks가 데이터·메타데이터·최적화·수명 주기를 관리→ Managed tableUnity Catalog의 기본 권장 선택
기존 클라우드 경로 사용, 외부 시스템도 파일에 직접 접근→ External tableDROP 시 메타데이터만 제거되고 원본 파일은 남음
외부 DB를 복사하지 않고 읽기 전용으로 조회→ Foreign table / Lakehouse Federation
현재 Spark 세션에서만 사용하는 중간 결과→ Temporary view

📥 수집과 스트리밍

클라우드 저장소에 계속 도착하는 새 파일을 확장성 있게 증분 수집→ Auto Loader
SQL로 파일을 재시도 가능하고 멱등하게 Delta 테이블에 적재→ COPY INTO이미 처리한 파일은 다시 실행해도 건너뜀
연속 데이터 처리, checkpoint, micro-batch→ Structured Streaming
Delta 테이블의 행 단위 insert·update·delete 변경 추적→ Change Data Feed활성화한 이후의 변경만 기록

⚙️ Delta 명령과 성능

조건에 따라 insert와 update를 한 번에 처리→ MERGE INTO
작은 파일 병합, 데이터 파일 레이아웃 개선→ OPTIMIZE
더 이상 참조되지 않는 오래된 데이터 파일 제거→ VACUUM
과거 버전 조회→ VERSION AS OF / TIMESTAMP AS OF
새 테이블, 높은 카디널리티, 변화하는 필터 패턴→ Liquid clustering현재 권장은 새 테이블에 partition·ZORDER보다 liquid clustering 우선. 기존 시험 문제는 ZORDER가 정답일 수 있음

🔐 Unity Catalog 권한

통합 권한, 데이터 계보, 감사, 검색·카탈로그→ Unity Catalog
테이블 읽기→ USE CATALOG + USE SCHEMA + SELECT
테이블 쓰기→ USE CATALOG + USE SCHEMA + MODIFY
스키마 안에 테이블 생성→ USE CATALOG + USE SCHEMA + CREATE TABLE
권한을 관리하지만 데이터를 읽거나 쓰지는 않음→ MANAGE소유권 이전까지 자동으로 허용하는 것은 아님

🛠️ 작업과 문제 해결

태스크 의존성, 예약, 재시도, 알림, 오케스트레이션→ Lakeflow Jobs
노트북 코드를 단계별 실행하고 변수 값 검사→ Databricks debugger / Variable Explorer
Stage, Task, shuffle, skew, executor 성능 분석→ Spark UI
클러스터 시작 실패나 인프라 이벤트 확인→ Cluster event log
용어 업데이트: 예전 문제의 Delta Live Tables (DLT)는 현재 Lakeflow Spark Declarative Pipelines, Workflows의 Job 기능은 Lakeflow Jobs라는 이름을 사용해. 시험 덤프의 구 명칭과 현재 문서 명칭을 둘 다 알아두는 게 좋아.
